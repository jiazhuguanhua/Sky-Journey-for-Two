# ✈️ Sky Journey for Two

**情侣专属的浪漫飞行棋网页游戏**

## 🎮 游戏简介

一款专为情侣设计的飞行棋游戏，结合真心话大冒险元素，让两个人的互动更加有趣和浪漫。

### ✨ 核心特性

- 🎯 **简单易玩**: 经典飞行棋规则，轻松上手
- 💕 **情侣专属**: 专为两人设计的任务和挑战
- 🎪 **真心话大冒险**: 丰富的任务库，增加游戏趣味性
- 🎨 **精美UI**: 现代化设计，丰富的动画效果
- 🔧 **可定制**: 支持任务格比例调整和任务类型选择
- 🎵 **音效支持**: 沉浸式游戏体验，自动播放BGM
- 📝 **任务编辑器**: 内置可视化任务库编辑器
- 🎭 **丰富动画**: 20+ 种动画效果，提升游戏体验
- 🔊 **多种音效**: 按钮、骰子、移动、任务、胜利等音效

### 🎲 游戏玩法

1. **设置玩家**: 输入昵称，选择头像（支持自动生成）
2. **选择任务类型**: 五种不同风格的任务库
3. **调整难度**: 自定义任务格在棋盘中的比例
4. **开始游戏**: 掷骰子移动，触发任务挑战
5. **完成挑战**: 真心话或大冒险，增进感情
6. **编辑任务**: 使用内置编辑器自定义任务内容

## 🚀 快速开始

### 环境要求

- Node.js 16+
- npm 或 yarn

### 安装与运行

```bash
# 克隆项目
git clone https://github.com/yourusername/sky-journey-for-two.git
cd sky-journey-for-two

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```

### 一键部署

#### GitHub Pages 自动部署 ⭐ 推荐

1. Fork 本项目到你的 GitHub 账户
2. 在仓库设置中启用 GitHub Pages
3. 推送代码到 main 分支，GitHub Actions 将自动构建和部署

```bash
git push origin main
```

#### 手动部署

```bash
# 使用部署脚本
./deploy.sh
```

或手动部署：

```bash
npm install
npm run build
npm run preview  # 本地预览
```

### 在线访问

访问 `http://localhost:5173` (开发模式) 或 `http://localhost:4173` (预览模式)

项目基于 Vite 构建，支持各种部署平台：

- **GitHub Pages**: 🔥 推荐，自动部署，免费托管
- **Vercel**: 零配置部署，速度快
- **Netlify**: 简单快捷
- **其他平台**: 支持任何静态托管服务

### 🚀 GitHub Pages 部署步骤

1. **Fork 项目**: 点击右上角 Fork 按钮将项目复制到你的账户
2. **启用 Pages**: 
   - 进入你的仓库设置 (Settings)
   - 找到 Pages 选项 (在左侧菜单中)
   - Source 选择 "GitHub Actions"
   - 保存设置
3. **推送代码**: 推送到 main 分支即可自动触发部署
4. **访问网站**: 部署完成后访问 `https://yourusername.github.io/Sky-Journey-for-Two/`

**⚡ 注意事项:**
- 首次部署可能需要 5-10 分钟
- 部署状态可在仓库的 Actions 标签页查看
- 每次推送 main 分支都会自动重新部署
- 项目已配置好所有必要的 GitHub Actions 工作流

**🔧 本地测试 GitHub Pages 构建:**

## 🔧 自定义配置

### 任务格比例
通过主页滑块调整任务格在棋盘中的占比（10%-70%）

### 添加背景音乐
将音频文件命名为 `bgm.mp3`、`bgm.mp4` 或 `bgm.wav` 放入 `public` 目录

### 任务库编辑器 ⭐ 新功能
点击主页"📝 任务编辑器"按钮进入可视化编辑界面：
- 🎯 选择任务库类型和类别（真心话/大冒险）
- ➕ 添加新任务或编辑现有任务
- 🔍 搜索和筛选任务
- 💾 保存到本地存储
- 📤 导出/导入任务库文件
- 🗑️ 删除不需要的任务

### 自定义任务（代码方式）
编辑 `src/data/taskLibrary.js` 文件添加自己的任务内容
